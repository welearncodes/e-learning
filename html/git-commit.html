Git Commit

Saving changes - git commit stores the current contents/snapshot of the index in a new commit along with a log message from the user describing the changes. Commited changes are "safe" snapshots of the project - Git will never change them unless ask it to.

Ex:

git commit -m "message"

Commit the staged snapshot, but instead of launching a text editor, use message to commit the message.
